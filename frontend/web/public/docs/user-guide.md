# 📖 OpenMake.Ai 사용자 가이드

**버전**: 1.5.0  
**최종 업데이트**: 2026-01-02

---

## 🎯 소개

**OpenMake.Ai**은 다양한 AI 모델을 활용하여 대화, 코드 작성, 문서 분석, 번역 등 다양한 작업을 수행할 수 있는 올인원 AI 어시스턴트입니다. Ollama 기반의 로컬 LLM과 Gemini 클라우드 모델을 모두 지원합니다.

## ✏️ 프롬프트 모드 (12종)

각 모드는 특정 작업에 최적화된 시스템 프롬프트를 사용합니다. 질문 내용에 따라 **자동으로 최적 모드가 선택**됩니다.

### 🟢 기본 모드

#### 1. Assistant (기본 어시스턴트)
- **용도**: 일반 대화, 질문 답변, 정보 제공
- **특징**: 친근하고 이해하기 쉬운 설명
- **예시**: "오늘 날씨 어때?", "커피 종류 알려줘"

#### 2. Reasoning (추론 전문가)
- **용도**: 복잡한 문제 해결, 수학적/논리적 분석
- **특징**: 단계별 사고 과정 표시 (Chain-of-Thought)
- **예시**: "이 수학 문제를 풀어줘", "왜 물은 100도에서 끓어?"

---

### 💻 개발 모드

#### 3. Coder (코드 전문가)
- **용도**: 프로그래밍, 디버깅, API 개발
- **특징**: 프로덕션 레벨 코드, 에러 핸들링 포함
- **예시**: "TypeScript로 REST API 만들어줘"

#### 4. Reviewer (코드 리뷰어)
- **용도**: 코드 품질 분석, 보안 취약점 검토
- **특징**: [Critical/Major/Minor] 수준별 이슈 분류
- **예시**: "이 코드 리뷰해줘", "성능 개선점 찾아줘"

#### 5. Generator (프로젝트 생성기)
- **용도**: 프로젝트 스캐폴딩, 보일러플레이트 생성
- **특징**: 완전한 디렉토리 구조와 초기 설정 파일
- **예시**: "Next.js 프로젝트 구조 만들어줘"

#### 6. Security (보안 전문가)
- **용도**: 보안 취약점 분석, 방어 전략 수립
- **특징**: OWASP 기준 위협 분석
- **예시**: "이 코드의 보안 취약점 찾아줘"

---

### 📝 콘텐츠 모드

#### 7. Writer (글쓰기 전문가)
- **용도**: 블로그, 이메일, 보고서 작성
- **특징**: 서사 구조와 독자 공감 중심
- **예시**: "블로그 포스트 써줘", "이메일 작성해줘"

#### 8. Explainer (기술 교육자)
- **용도**: 복잡한 개념을 쉽게 설명
- **특징**: 비유와 예시를 활용한 설명
- **예시**: "양자컴퓨터가 뭐야?", "블록체인 설명해줘"

#### 9. Translator (번역 전문가)
- **용도**: 다국어 번역, 문화적 맥락 해설
- **특징**: 원문 뉘앙스 보존
- **예시**: "이 문장 영어로 번역해줘"

---

### 🔬 분석 모드

#### 10. Researcher (정보 분석가)
- **용도**: 리서치, 데이터 조사, 트렌드 분석
- **특징**: Exa Search 도구 연동, 출처 명시
- **예시**: "최신 AI 트렌드 조사해줘"

#### 11. Consultant (전략 컨설턴트)
- **용도**: 비즈니스 전략, 문제 해결, 로드맵 수립
- **특징**: 리스크 분석과 실행 계획 포함
- **예시**: "스타트업 창업 전략 세워줘"

---

### 🤖 에이전트 모드

#### 12. Agent (실행형 에이전트)
- **용도**: 외부 도구 호출, 자동화 작업
- **특징**: ReAct 패턴 (Thought → Action → Observation)
- **예시**: "웹에서 최신 뉴스 검색해줘"

## 🔧 MCP 도구

외부 서비스와 연동하여 LLM의 능력을 확장합니다.

### 사용 가능한 도구

| 도구 | 기능 |
|------|------|
| 🔍 웹 검색 | Google/DuckDuckGo/Naver 검색 |
| 🐙 GitHub | 레포지토리 검색, 이슈 관리, 코드 조회 |
| 📊 Exa Search | 학술/기술 문서 검색 |
| 📄 PDF 분석 | 문서 요약 및 Q&A |
| 🖼️ 이미지 분석 | Vision 기반 이미지 인식 |

### 활성화 방법
1. **설정** → **MCP 도구** 메뉴 이동
2. 원하는 도구 토글 ON
3. 필요한 API 키 설정 (GitHub, Exa 등)

---

## 📄 문서 분석

### 지원 형식
- PDF, DOCX, TXT, MD
- 이미지 (PNG, JPG) - OCR 지원
- 엑셀 (XLSX) - 데이터 추출

### 사용 방법
1. 채팅 입력창의 📎 버튼 클릭
2. 파일 업로드 (최대 100MB)
3. "이 문서 요약해줘" 또는 "3페이지 내용 설명해줘"

## ⚙️ 설정 옵션

### 일반 설정
- **테마**: 다크/라이트 모드
- **언어**: 한국어/영어

### AI 설정
- **기본 모델**: Gemini 3 Pro / Flash
- **Temperature**: 창의성 수준 (0.1~1.0)
- **Sequential Thinking**: 단계별 사고 활성화

### 데이터 관리
- **대화 내보내기**: JSON/CSV 형식
- **히스토리 삭제**: 선택적/전체 삭제

## 💡 팁 & 트릭

### 효과적인 프롬프트 작성
```
✅ 좋은 예시:
"TypeScript로 사용자 인증 API를 작성해줘. 
JWT 토큰을 사용하고 에러 처리를 포함해줘."

❌ 나쁜 예시:
"API 만들어줘"
```

### 자동 모드 선택 활용
- "번역해줘" → 자동으로 Translator 모드
- "코드 리뷰해줘" → 자동으로 Reviewer 모드
- "왜 그런거야?" → 자동으로 Reasoning 모드

### 웹 검색 활용
- 시사 관련 질문에 자동 웹 검색 적용
- "오늘", "최근", "현재" 등 키워드 포함 시 활성화

## ❓ FAQ

### Q: 어떤 모델을 사용하나요?
**A**: 기본적으로 Gemini 3 Flash/Pro를 사용하며, 설정에서 변경할 수 있습니다.

### Q: 대화 내용은 저장되나요?
**A**: 네, 로그인 사용자의 대화는 서버에 저장되며 히스토리 메뉴에서 확인할 수 있습니다.

### Q: 파일 업로드 용량 제한은?
**A**: 최대 100MB입니다.

### Q: 프롬프트 모드를 수동으로 변경할 수 있나요?
**A**: 네, 사이드바의 프롬프트 모드 섹션에서 원하는 모드를 클릭하세요.
