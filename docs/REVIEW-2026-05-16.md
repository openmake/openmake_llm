## 2. 아키텍처 & 기술 부채

### 2.1 No-Hardcoding 정책 위반 (검증됨)

**`model.routes.ts:42-50` — `KNOWN_MODELS` 하드코딩** ✅ 확인
```ts
const KNOWN_MODELS: Record<string, ...> = {
    openrouter: [
        { id: 'openai/gpt-5', displayName: 'GPT-5', ... },
        { id: 'anthropic/claude-opus-4.5', ... },
        { id: 'google/gemini-2.5-pro', ... },
```

**`config/runtime-limits.ts:23, 39-50, 64-74` — 매직 넘버 (env override 미지원)** ⚠️ 부분 확인

## 3. 보안 & 인증

### 3.1 진짜 Critical 1건

**`chat-renderer.js:325` `insertAdjacentHTML` thinking trace 재삽입 미살균** ✅ 확인
```js
if (thinkingTraceHtml) {
    content.insertAdjacentHTML('afterbegin', thinkingTraceHtml);
}
```
- `thinkingTraceHtml`은 직전에 `existingThinkingTrace.outerHTML`로 캡처된 값을 다시 삽입. 캡처 시점에 이미 sanitize 됐다면 안전하지만, 만약 어딘가에서 미살균 상태로 DOM에 들어왔다면 재삽입으로 X SS 트리거.
- **권장**: `purifyHTML()` 한 번 더 통과시키거나, `appendChild`로 노드 자체를 재사용.

### 3.2 진짜 High 2건

**`token-crypto.ts:59-65` — `TOKEN_ENCRYPTION_KEY` 미설정 시 평문 fallback** ✅ 확인
```ts
const key = getKey();
if (!key) return plaintext; // no-op fallback (개발환경 또는 키 미설정)
```
- 운영 환경에서 키 누락 시 silent fallback, 즉 OAuth 토큰이 DB에 평문 저장됨.
- **권장**: `NODE_ENV === 'production' && !key` 시 throw. dev는 현재 동작 유지.

**`mcp/filesystem.ts:134-181` — `fs_read_file`에 symlink 차단 없음** ⚠️ 부분 확인 (서브에이전트 주장, 코드 미확인)
- `validatePathWithinSandbox()`는 호출되지만 `fs.realpath()`로 symlink 해소 후 재검증하지 않으면 sandbox 탈출 가능.
- **권장**: 진단 후 patch.

### 3.3 Medium

- `auth/auth-core.ts` — refresh token rotation 없음. 누출된 refresh token 무기한 유효.
- `auth/oauth-provider.ts:247` — OAuth state 만료 10분 + clock skew 영향 가능.
- `routes/api-keys.routes.ts` — `requireAuth`만 있고 scope/role middleware 없음. 핸들러 내 role 체크에 의존.
- `middlewares/index.ts:122` — `Access-Control-Allow-Credentials: true` 글로벌. CORS_ORIGINS 미설정 시 빈 화이트리스트로 안전하지만 misconfig 시 CSRF 위험 증폭.

### 3.4 운영 위생

- `npm audit --production` 실행 권한 없어 수행 못함 → 사용자가 직접 `npm audit --omit=dev` 실행 권장.
- `.env`는 미커밋 확인됨 (FP-1). 단, **백업/덤프 시 노출 위험은 별개**. `.env.example`에 dummy 값만 있는지 점검 필요.

## 4. 성능 & 확장성

### 4.1 진짜 High 3건

**`sockets/handler.ts:100-106` — WebSocket 브로드캐스트 backpressure 없음** ✅ 확인
- `cluster_event`를 `this.broadcast({...})`로 전체 송출, `ws.send()`에 await/콜백 미부착.
- 슬로우 클라이언트의 TCP window full → `send()` 큐 누적 → Node 메모리/이벤트 루프 stall.
- **권장**: 클라이언트별 `bufferedAmount` 체크, 임계 초과 시 `ws.terminate()` 또는 backpressure-aware queue.

**`config/timeouts.ts:49-58` — pg Pool `max=20`, `IDLE_TIMEOUT=30s`** ⚠️ 부분 확인 (실제 사용 패턴 측정 필요)
- 동시 요청 20+ 시 connection 대기 → 응답 latency 비례 증가.
- **권장**: pg Pool 메트릭(`pool.totalCount`, `pool.idleCount`, `pool.waitingCount`)을 5초 주기로 기록 → 임계 초과 시 경고.

**`eng.traineddata` (5MB) + `kor.traineddata` (2MB) repo root commit** ✅ 확인 (`ls -la`)
- Tesseract 학습 데이터는 lazy download (예: postinstall script로 첫 OCR 호출 시 다운로드) 또는 CDN 호스팅.
- 현재는 매 clone에 강제 다운로드.

### 4.2 진짜 Medium

- **`ecosystem.config.js`**: `max_memory_restart: 1G`, `log_type: 'json'` 설정됐지만 rotation 정책 없음. ✅ 확인.
  - `pm2-logrotate` 모듈 + `max_size: '10M'`, `retain: 30`, `compress: true` 추가 권장.
- **`conversation-sessions.ts:148-156`**: `getAllSessions(limit=100)`, `getSessionsByUserId(limit=50)` — 한도가 있지만 **`loadMessagesForSessions()`가 세션별 LIMIT 없이 모든 메시지 join**. 5k+ 메시지 사용자 스파이크.
- **`semantic-cache.ts:160-177`**: LRU eviction 발생 시 로그/메트릭 없음 → 캐시 thrashing 진단 불가.
- **`cluster/config.ts:38-46`**: `loadClusterConfig()` 부팅 시 `readFileSync` — 네트워크 마운트 등에서 startup 지연 가능. 운영 환경에서 문제 없으면 P3.

### 4.3 마이그레이션 인덱스 (014, 015)

- `014_conversation_audit_log.sql`: `(user_id, created_at DESC)` 복합 인덱스 누락 → 사용자별 timeline 쿼리 성능 저하.
- `015_conversation_debug_queue.sql`: `expires_at` 인덱스가 graceful-skip 패턴(`DO + EXCEPTION`)으로 작성되어 운영 DB의 owner mismatch 시 인덱스 미생성 가능.
- **권장**: 마이그레이션 후 운영 DB에서 `\d+ table_name`으로 인덱스 실제 존재 확인하는 검증 스크립트 추가.
