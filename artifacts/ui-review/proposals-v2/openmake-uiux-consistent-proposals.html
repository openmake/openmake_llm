V2 · 페이지군별 유지형 개선 Docs / Guide / Documents

문서형 페이지는 읽기/실행 흐름을 보강

사용 가이드와 API 문서는 현재의 카드, 보더, 퍼플 액센트를 유지하면서 탐색 피로를 낮춰야 합니다. 문서 관리는 빈 상태를 유지하되 업로드 이후 다음 행동을 명확히 제안합니다.

사용 가이드읽기 개선

Sticky 목차

좌측 사이드바는 유지하고, 본문 내부에 섹션 목차만 얇게 고정합니다.

빠른 시작

상단에 “3분 시작”, “모드별 예시”, “문서 분석” 탭을 추가합니다.

긴 문서를 전부 읽게 하지 않고, 사용자가 목적별로 바로 이동하도록 만듭니다.

API 문서실행 개선

Copy/Test CTA

Base URL, cURL, SDK 예제에 복사/테스트 버튼을 고정합니다.

Docs Mode

앱 사이드바는 유지하되 문서 내부 TOC 폭을 줄여 본문 폭을 확보합니다.

API 문서는 읽는 문서가 아니라 바로 검증하는 콘솔에 가까워야 합니다.

문서 관리빈 상태 개선

처리 상태

업로드, 파싱, 인덱싱, 질문 가능 상태를 단계로 표시합니다.

질문 시작

업로드 완료 후 “이 문서로 질문하기” CTA를 즉시 보여줍니다.

현재의 업로드 박스 스타일은 유지하고, 작업 완료 이후 행동만 추가합니다.

P3
