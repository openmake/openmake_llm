## 1. 다층 프롬프트 시스템 개념

### 🏗️ 다층 구조란?
여러 개의 프롬프트 층(Layer)이 순차적으로 조합되어 최종 시스템 프롬프트를 구성하는 아키텍처.

```
기본 에이전트 프롬프트
    ↓ 추가
DB 스킬 / 파일 기반 프롬프트  
    ↓ 추가
컨텍스트 엔지니어링 (4-Pillar Framework)
    ↓ 최종
완성된 시스템 프롬프트 (10개 층)
```

### 📊 10개 층 구조 상세

#### Phase 1: STATIC (정적 - 캐시 최적화)
1. **[Identity] 역할 정의** - 페르소나 확립
2. **[Rules] 제약 조건** - 보안/언어/행동 규칙
3. **[Format] 출력 형식** - 응답 구조 지정
4. **[Process] 소프트 인터락** - 사고 프로세스 강제
5. **[Reinforcement] 최종 리마인더** - Double Recency (핵심)

#### Phase 2: DYNAMIC (동적 - 요청별 변경)
6. **[Context] 메타데이터** - 날짜, 세션, 모델 정보
7. **[Knowledge] RAG 컨텍스트** - 검색된 참조 문서
8. **[Examples] Few-shot 예시** - 학습 샘플
9. **[Agentic] 추가 동적 섹션** - 특수 요구사항
10. **[Task] 과업 목표** - 구체적 실행 목표

## 2. 언어 감지 및 처리 로직

### 🔍 detectLanguageForMetadata() 분석

**파일**: `backend/api/src/chat/context-engineering.ts:578-593`

```typescript
export function detectLanguageForMetadata(text: string): 'ko' | 'en' | 'mixed' {
    const koreanRegex = /[가-힣]/g;        // 한글만 감지
    const englishRegex = /[a-zA-Z]/g;      // 영문자만 감지
    
    const koreanMatches = (text.match(koreanRegex) || []).length;
    const englishMatches = (text.match(englishRegex) || []).length;
    
    const total = koreanMatches + englishMatches;
    if (total === 0) return 'en';          // 기본값: 영어
    
    const koreanRatio = koreanMatches / total;
    
    if (koreanRatio > 0.7) return 'ko';    // 한국어 70% 이상
    if (koreanRatio < 0.3) return 'en';    // 한국어 30% 미만  
    return 'mixed';                        // 30-70% 혼합
}
```

### 🌐 언어별 처리 결과

| 입력 언어 | 감지 결과 | 응답 언어 | 이유 |
|---------|---------|---------|------|
| "안녕하세요" | `ko` | **한국어** | 한글 문자 70% 이상 |
| "Hello world" | `en` | **영어** | 영문자만 존재 |
| "¿Hola cómo estás?" | `en` | **영어** | 스페인어 → 영문자로 분류 |
| "Bonjour comment allez-vous?" | `en` | **영어** | 프랑스어 → 영문자로 분류 |
| "こんにちは" | `en` | **영어** | 일본어 → 문자 감지 안됨 |
| "你好" | `en` | **영어** | 중국어 → 문자 감지 안됨 |

## 6. 우선순위 및 덮어쓰기 규칙

### ⚡ 프롬프트 우선순위

1. **기본값**: 한국어 응답 (모든 기본 프롬프트)
2. **감지**: 사용자 입력 언어 자동 감지
3. **최종**: `final_reminder`의 언어 규칙이 **모든 것을 덮어씀**

### 🔄 덮어쓰기 메커니즘

```
기본 프롬프트: "한국어로 응답"
       ↓
언어 감지: 스페인어 → 'en'
       ↓
최종 확인: "영어로 답변" ← 승리!
```

## 7. 시스템 한계 및 개선점

### ❌ 현재 한계

1. **이진 분류**: 한국어-영어만 지원
2. **라틴 문자 언어**: 모두 영어로 오분류
3. **비라틴 문자 언어**: 감지 불가 → 기본값 영어

### 💡 개선 방향

1. **다국어 감지 라이브러리** 도입 (예: franc, langdetect)
2. **언어별 응답 템플릿** 확장
3. **사용자 언어 설정** 명시적 선택 옵션
4. **언어 감지 정확도** 향상

## 9. 핵심 설계 원칙

### 🎯 위치 공학 (Position Engineering)

1. **Primacy Effect**: 중요한 정보를 앞에 배치
2. **Recency Effect**: 최종 규칙을 뒤에 배치 (덮어쓰기)
3. **Double Recency**: `final_reminder`로 재강조

### 🏎️ 캐시 최적화

1. **정적 섹션**: 요청 간 동일 → 캐시 활용
2. **동적 섹션**: 요청별 변경 → 실시간 생성

### 📐 4-Pillar Framework

1. **Role** (정체성): 누구인가?
2. **Constraints** (제약): 어떻게 행동해야 하는가?
3. **Goal** (목표): 무엇을 달성해야 하는가?
4. **OutputFormat** (형식): 어떤 형태로 답해야 하는가?
